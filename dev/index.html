This is a Julia package for modeling and solving Newsvendor Problems.

Quick start

The package is not yet listed and must be retrieved from the Github repo.

julia> import Pkg
julia> Pkg.add(url="https://github.com/frankhuettner/NewsvendorModel.jl")

After installation, it can be loaded with the usual command.

julia> using NewsvendorModel

Moreover, you need to load the distributions package.

julia> using Distributions

You can now define a model with the command NVModel(...) using the following required arguements (in that order):

  • unit production cost
  • unit selling price
  • demand distribution

Solve for optimal quanitity and obtain key metrics with the optimize() function.

Example

Consider a scenario with

  • unit cost = 1
  • unit price = 5
  • demand that follows a normal distribution with
    • mean = 90
    • standard deviation = 30

The model is now defined and stored in the variable nvm as follows:

julia> nvm = NVModel(1, 5, Normal(90, 30))

Julia shows the model data:

Data of the Newsvendor Model
 * Unit cost: 1.00
 * Unit selling price: 5.00
 * Demand distribution: Normal{Float64}(μ=90.0, σ=30.0)
 * Unit salvage value: 0.00
 * Unit backlog penalty: 0.00
 * Fixed cost: 0.00

Next, you can solve the model like so:

julia> res = optimize(nvm)

This gives the following output:

=====================================
Results of maximizing expected profit
 * Optimal quantity: 116
 * Expected profit: 317.99
=====================================
This is a consequence of
 * Cost of underage: 4.00
 * Cost of ovderage: 1.00
 * The critical fractile: 0.80
 * Usage of round-up rule: true
-------------------------------------
Ordering the optimal quantity yields
 * Expected sales: 86.80
 * Expected lost sales: 3.20
 * Expected leftover: 29.20
 * Expected backlog penalty: 0.00
-------------------------------------

Moreover, you have stored the result in the varial res. Reading the data from the stored result is straight-forward:

julia> q_opt(res)
116
julia> profit(res)
317.9926337188953

Analogously, underage_cost(res), overage_cost(res), critical_fractile(res), roundup_rule(res), sales(res), lost_sales(res), leftover(res), penalty(res), read the other information the stored result res. The model that was solved can be retrieved with nvmodel(res).

Further options

Consider a sligth variation of the example above.

Demand distribution

Let us rule out demand below zero, i.e., let us truncate the distribution at 0. Working with distributions is very convinient in Julia. We can define our distribution of choice as follows: 

julia> my_distr = truncated(Normal(90, 30), 0, Inf)
Truncated(Normal{Float64}(μ=90.0, σ=30.0), range=(0.0, Inf))

Note that this package shall work with any univariate distribution from the Distributions.jl package.

The model is now defined and stored in the variable nvm2 as follows:

julia> nvm2 = NVModel(1, 5, my_distr)
Data of the Newsvendor Model
 * Unit cost: 1.00
 * Unit selling price: 5.00
 * Demand distribution: Truncated(Normal{Float64}(μ=90.0, σ=30.0), range=(0.0, Inf))
 * Unit salvage value: 0.00
 * Unit backlog penalty: 0.00
 * Fixed cost: 0.00

(Alternatively, we could define it at once as above: nvm2 = NVModel(1, 5, truncated(Normal(90, 30), 0, Inf)))

Salvage value, backlog penalty, and fixed cost

Further, suppose that

  • an unsold unit can be salvaged for 0
  • not serving a demanded unit not only misses out on profit but also comes at a backlog penalty of 2
  • the operations require a fixed cost of 100

The model is now specified as follows:

julia> nvm2 = NVModel(1, 5, my_distr, salvage = 0.3, backlog = 2, fixcost = 100)
Data of the Newsvendor Model
 * Unit cost: 1.00
 * Unit selling price: 5.00
 * Demand distribution: Truncated(Normal{Float64}(μ=90.0, σ=30.0), range=(0.0, Inf))
 * Unit salvage value: 0.30
 * Unit backlog penalty: 2.00
 * Fixed cost: 100.00

Round-up rule

Finally, let us assume that we do not want the optimal integer quantity but the exact real number. To this end, we will call the optimize function with the additional argument roundup_rule=false.

You solve the model like so:

julia> optimize(nvm2, roundup_rule=false)
=====================================
Results of maximizing expected profit
 * Optimal quantity: 127.72
 * Expected profit: 224.19
=====================================
This is a consequence of
 * Cost of underage: 6.00
 * Cost of ovderage: 0.70
 * The critical fractile: 0.90
 * Usage of round-up rule: false
-------------------------------------
Ordering the optimal quantity yields
 * Expected sales: 88.64
 * Expected lost sales: 1.50
 * Expected leftover: 39.08
 * Expected backlog penalty: 2.99
-------------------------------------
